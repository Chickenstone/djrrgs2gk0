# 东兴便民通 - 项目维护与修改指南

如果您需要对“东兴便民通”小程序的内容进行修改或日常维护，请参考以下指南。本项目采用了**前端页面+云端数据库**的架构，因此修改内容分为两类：**修改数据**和**修改界面**。

---

## 1. 如何修改“动态数据”？
*(如：美食列表、景点列表、文创商品)*

这些数据已经接入了**腾讯云开发 (CloudBase)**。如果您需要增加、删除或修改这些内容，**完全不需要修改代码**。

**操作步骤：**
1. 登录 [腾讯云开发控制台](https://console.cloud.tencent.com/tcb)。
2. 进入您的环境，点击左侧菜单的 **“数据库”**。
3. 找到对应的集合 (Collection)：
   - 要改美食：进入 `restaurants` 集合。
   - 要改景点：进入 `spots` 集合。
   - 要改文创商品：进入 `products` 集合。
4. 在记录列表中，点击对应的记录进行“编辑”，或者点击“添加记录”新增一条。修改保存后，小程序端会自动同步最新数据。

---

## 2. 如何修改“静态界面与文案”？
*(如：首页的轮播图、政务大厅的办事指南、各个按钮的文字)*

这些内容是写死在前端代码里的。如果您需要修改，请打开代码编辑器（如 Trae / VS Code），找到对应的文件进行修改。

### 📍 核心页面修改定位表

#### 🏠 首页 (`src/pages/home/index.tsx`)
- **想换轮播图图片**：搜索代码中的 `CAROUSEL_IMAGES` 数组，替换里面的 `url` 链接。
- **想改“为您推荐”的内容**：搜索 `RECOMMENDATIONS` 数组，修改 `title`、`desc` 或 `image`。

#### 🏛️ 政务模块 (`src/pages/government/index.tsx`)
- **想改办事指南的列表**：搜索 `SERVICES` 数组，修改 `title`（标题）或 `desc`（描述）。

#### 🛠️ 一条龙服务 (`src/pages/service/index.tsx`)
- **想改服务包的价格和内容**：搜索 `PACKAGES` 数组，修改里面的 `title`、`price`、`features` 等字段。

#### 👤 个人中心 (`src/pages/user/index.tsx`)
- **想改底部的版权声明**：拉到代码最底部，找到 `@chickenstone` 相关的文本直接修改。

---

## 3. 如何修改“样式与颜色”？

本项目使用 **Tailwind CSS** 来编写样式。如果您想改变某个按钮或背景的颜色，只需在对应的 `.tsx` 文件中修改元素的 `className`。

*   **蓝色系**：`bg-blue-500`, `text-blue-600`
*   **橙色系**：`bg-orange-500`, `text-orange-500` (美食模块常用)
*   **绿色系**：`bg-green-500` (攻略模块常用)
*   **红色系**：`bg-red-600` (文创模块常用)

例如，把一个蓝色按钮改成红色：
```tsx
// 修改前
<button className="bg-blue-600 text-white">确定</button>

// 修改后
<button className="bg-red-600 text-white">确定</button>
```

---

## 4. 修改代码后如何发布上线？

如果您修改了任何 **代码文件（.tsx）**，需要重新打包并上传到腾讯云，用户才能看到更新。

1. 在终端运行打包命令：
   ```bash
   npm run build
   ```
2. 登录腾讯云控制台 -> 静态网站托管。
3. 将项目目录下新生成的 `dist` 文件夹内的**所有内容**，拖拽上传并覆盖旧文件。
4. （可选）在静态网站托管的“缓存配置”中点击“刷新缓存”，确保用户立即看到最新版本。

# 东兴便民通 - 微信小程序项目

本项目是一个为广西东兴市量身定制的综合性生活服务/跨境旅游小程序前端项目。涵盖了政务、美食、旅游攻略、一条龙服务和文创抽奖等五大核心模块。

| **政务大厅** | `src/pages/government/index.tsx` | 展示东兴市政府相关的办事指南（如：越南落地签申请指南、电子网格证申领等）。 |

# 仓库关联与部署架构说明

本文档用于说明当前私有仓库（`dxapp`）与外部公开部署仓库之间的关系与同步规范。

## 核心原则

1. **唯一真理来源 (Single Source of Truth)**
   - 本私有仓库 **`dxapp`** 是项目的唯一核心代码库。
   - 所有的新功能开发、Bug 修复、文档更新以及配置变更，都**必须**首先在 `dxapp` 仓库中进行并提交。
   - 该仓库包含项目的所有隐私信息、架构文档和历史开发记录。

2. **公开部署仓库 (Deployment Repository)**
   - 外部公开仓库名称：**`djrrgs2gk0`**
   - **用途**：该公开仓库**仅用于**与外部云平台（如腾讯云 Webify 等）进行连接与自动化静态网站部署。
   - **隐私保护**：
     - 为了防止项目被未经授权的人员轻易搜索或识别，该仓库采用了一串 10 位的随机码（`djrrgs2gk0`）作为名称，以保证一定的隐私性。
     - 该公开仓库中**不包含**任何项目的详细 Markdown 说明文档（如 `README.md`、开发文档等），仅保留纯粹的运行代码。

3. **同步规范**
   - 公开仓库 `djrrgs2gk0` 内的所有内容和代码版本，应**无条件以私有的 `dxapp` 仓库为准**。
   - 严禁直接在公开仓库 `djrrgs2gk0` 中进行代码修改或直接提交。
   - 每次在 `dxapp` 仓库完成功能开发并测试通过后，应通过代码同步机制（或手动推送）将构建所需的最新源码同步至 `djrrgs2gk0` 仓库，进而触发外部云平台的自动构建与部署。

# 腾讯云静态网站托管部署指南

既然我们已经在使用腾讯云开发 (CloudBase) 的数据库服务，那么最完美、最原生的国内部署方案就是：**腾讯云开发 - 静态网站托管**。

它不仅在国内访问速度极快（自带腾讯云 CDN），而且与我们现在的数据库环境是同一个体系，管理起来非常方便。

## 部署步骤

### 1. 构建前端生产代码
在终端运行以下命令，将我们的 React 代码打包成最终的静态文件：
```bash
npm run build
```
执行完毕后，你的项目根目录下会生成一个 `dist` 文件夹。

### 2. 在腾讯云开启“静态网站托管”
1. 登录 [腾讯云开发控制台](https://console.cloud.tencent.com/tcb)。
2. 进入你之前创建的环境。
3. 在左侧菜单栏找到 **“静态网站托管”**。
4. 点击 **“开通”**（如果是首次使用，需要几分钟的初始化时间）。

### 3. 上传文件
开通成功后，你会看到一个文件管理的界面。
1. 点击 **“上传文件”**（或直接拖拽）。
2. 选择你本地电脑上 `dxapp/dist` 文件夹里面的**所有文件和文件夹**上传上去。（注意：是上传 `dist` 里面的内容，而不是上传 `dist` 这个文件夹本身）。

### 4. 获取访问域名
上传完成后，在“静态网站托管”页面的上方，你会看到一个 **“基础配置” -> “默认域名”**（类似 `xxxx.tcloudbaseapp.com`）。
点击这个域名，你就可以在国内流畅地访问你的“东兴便民通”应用了！

## 终极方案：GitHub Actions 自动持续部署 (CI/CD)

为了实现“代码一推，网站自动更新”，本项目已配置好 GitHub Actions 自动化工作流。只要您将代码推送到 GitHub 仓库的 `main` 分支，系统就会自动帮您打包并部署到腾讯云。

### 准备工作（仅需配置一次）

1. **获取腾讯云 API 密钥**
   - 登录腾讯云控制台。
   - 鼠标悬停在右上角的头像处，点击 **“访问管理” (CAM)**。
   - 在左侧菜单选择 **“访问密钥” -> “API 密钥管理”**。
   - 点击“新建密钥”，您将获得一对 `SecretId` 和 `SecretKey`。**请妥善保管，不要泄露给他人。**

2. **在 GitHub 仓库中配置 Secrets**
   - 打开您的项目 GitHub 仓库页面。
   - 点击上方的 **“Settings”**（设置）选项卡。
   - 在左侧菜单栏找到 **“Secrets and variables” -> “Actions”**。
   - 点击右上角的绿色按钮 **“New repository secret”**。
   - 依次添加以下三个 Secret：
     - Name: `TENCENT_SECRET_ID` / Value: 填入您刚才获取的 SecretId
     - Name: `TENCENT_SECRET_KEY` / Value: 填入您刚才获取的 SecretKey
     - Name: `TCB_ENV_ID` / Value: 填入您的云开发环境 ID（例如：`dxbmapp-5gp7z3tpe88b9050`）

### 如何触发自动部署？

配置完成后，您在本地修改完代码后，只需要执行常规的 Git 提交流程：

```bash

git push origin main
```

代码推送到 `main` 分支后，您可以去 GitHub 仓库的 **“Actions”** 选项卡查看部署进度。通常在 1-2 分钟内，您的网站就会自动更新为最新版本！

# 东兴便民通 - 代码维基 (Code Wiki)

## 1. 整体项目架构
本项目是一个采用“移动端优先”策略构建的 Web 应用，旨在模拟微信小程序的交互体验。它为广西东兴市量身定制，是一个涵盖政务、美食、旅游攻略、一条龙服务和文创抽奖等功能的综合性生活服务与跨境旅游平台。

**技术栈:**
- **前端框架:** React 18
- **构建工具:** Vite
- **路由管理:** React Router DOM v7 (单页应用架构 SPA)
- **样式方案:** Tailwind CSS (实用类优先，响应式设计), Lucide React (矢量图标库)
- **后端服务 (BaaS):** 腾讯云开发 CloudBase (提供 NoSQL 云数据库和身份验证能力)
- **状态管理:** 使用 React Hooks 进行局部状态管理，通过 `@cloudbase/js-sdk` 进行全局用户会话管理。

## 2. 核心模块与职责
应用程序按业务领域划分为多个独立模块，可通过底部导航栏 (`MobileLayout`) 切换访问。

- **首页模块 (`/src/pages/home`)**:
  - 应用程序的主入口。
  - 提供全局搜索框、顶部轮播图、各功能模块的快速导航入口，以及信息流推荐列表。
- **政务模块 (`/src/pages/government`)**:
  - 静态展示东兴市政府相关的办事指南，例如：越南落地签申请指南、电子网格证申领等。
- **美食模块 (`/src/pages/food`)**:
  - 展示东兴本地特色美食及餐饮信息。
  - 动态从腾讯云开发数据库 (`restaurants` 集合) 获取数据。
  - 包含餐厅分类过滤功能，以及基于“拥挤度”的智能标签和热力图引导提示。
- **旅游攻略模块 (`/src/pages/travel`)**:
  - 为金滩、京族三岛等本地景点提供智慧旅游指南。
  - 动态从腾讯云开发数据库 (`spots` 集合) 获取景点数据。
  - 集成实时天气、景点客流监控看板，以及 AI 动态行程规划入口。
- **一条龙服务模块 (`/src/pages/service`)**:
  - 提供透明的跨境旅游服务包展示（如：跨境一日游、金牌买手导购等）。
  - 需要用户通过腾讯云开发进行身份验证（登录）后方可使用。
  - 处理用户预约表单的提交，并将数据写入腾讯云开发数据库的 `bookings` 集合。
- **文创商城模块 (`/src/pages/culture`)**:
  - 展示本地特色文创商品及非遗纪念品。
  - 动态从腾讯云开发数据库 (`products` 集合) 获取商品数据。
  - 包含游戏化互动元素，如非遗景点打卡任务以获取积分兑换奖励。
- **个人中心模块 (`/src/pages/user`)**:
  - 管理用户的个人资料和应用状态。
  - 处理腾讯云开发的身份认证逻辑（模拟微信一键登录/退出登录/匿名静默登录）。
  - 展示用户的订单状态面板、各项服务（如政务、电子凭证）的快捷入口。

## 3. 核心类与函数说明

- **`src/utils/cloudbase.ts`**:
  - `app`: 腾讯云开发 (CloudBase) 实例，通过环境变量 `VITE_CLOUDBASE_ENV_ID` 初始化。
  - `auth`: CloudBase 身份验证实例 (`app.auth()`)，开启了本地持久化 (`persistence: 'local'`)。
  - `db`: CloudBase 数据库实例 (`app.database()`)，用于执行数据的增删改查操作。
  - `signInAnonymously()`: 辅助函数。当用户未主动登录时，静默执行匿名登录。这确保了即便是游客身份，也能成功获得对云数据库的读取权限（需配合数据库的安全规则配置）。
- **`src/components/layout/MobileLayout.tsx`**:
  - `MobileLayout`: 核心的页面布局包装器组件。它使用 React Router 的 `<Outlet />` 来渲染当前匹配的页面内容，并固定在底部渲染一个跨页面的导航栏 (`nav`)。
- **`src/App.tsx`**:
  - 根组件，配置了 `BrowserRouter`。它负责将所有的 URL 路径路由映射到 `MobileLayout` 下对应的具体页面组件。

## 4. 依赖关系
- **UI 与 DOM 渲染**: 依赖 `react` 和 `react-dom`。`react-router-dom` 拦截浏览器的 URL 变化，实现在不刷新页面的情况下切换模块视图。
- **样式构建流水线**: `tailwindcss` 协同 `postcss` 和 `autoprefixer` 处理 CSS 实用类。在组件中大量使用 `clsx` 和 `tailwind-merge` 来动态合并并应用条件 CSS 类名，避免样式冲突。`lucide-react` 用于向 DOM 中直接注入 SVG 图标。
- **后端连接层**: `@cloudbase/js-sdk` 被深度集成到各个业务组件中 (`Food`, `Travel`, `Culture`, `Service`, `User`)。它让前端可以直接与腾讯云通信，替代了传统的 REST API 模式，用于数据读取 (`db.collection().get()`)、数据写入和用户会话管理 (`auth.getLoginState()`)。
- **开发与构建工具**: `vite` 负责驱动本地开发服务器（提供热更新 HMR）和生产环境的打包构建。`typescript` 提供了包括 CloudBase 数据模型在内的强类型检查。

## 5. 项目运行指南

1. **环境准备**: 确保你的计算机上已安装 Node.js (推荐 v18 或以上版本)。
2. **安装依赖**:
   在项目根目录下执行：
   ```bash
   npm install
   ```
3. **环境变量配置**:
   在项目根目录创建一个名为 `.env.local` 的文件，并填入你的腾讯云开发环境 ID：
   ```env
   VITE_CLOUDBASE_ENV_ID=你的环境ID
   ```
4. **云数据库设置**:
   进入腾讯云开发控制台，创建以下四个集合（Collection）以使页面能正常加载和写入数据：
   - `restaurants` (权限：所有用户可读)
   - `spots` (权限：所有用户可读)
   - `products` (权限：所有用户可读)
   - `bookings` (权限：所有用户可读，仅创建者及管理员可写)
5. **启动开发服务器**:
   ```bash
   npm run dev
   ```
   启动后，在浏览器中访问 `http://localhost:5173`。*(提示：建议在浏览器的开发者工具中开启“设备工具栏/移动端视图”，以获得最佳的响应式预览体验)。*
6. **生产环境构建**:
   ```bash
   npm run build
   ```
